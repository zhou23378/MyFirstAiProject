### 11.2 对标系统核心设计原则

#### Shopify Polaris（商户后台黄金标准）

| 原则 | 说明 |
|------|------|
| **Progressive Disclosure** | 默认只显示最常用操作，高级选项折叠在"更多"中，减少 60% 视觉噪音 |
| **Contextual Actions** | 操作按钮不出现在全局操作栏，而出现在数据行/卡片的上下文中 |
| **Strong Page Header** | 每页必有标题 + 面包屑 + 主操作按钮（PageHeader 模式，100% 覆盖） |
| **Card-First Layout** | 每个业务模块独立成卡片，卡片内包含：数据摘要 + 快捷操作 + 深入链接 |
| **8px Spacing Grid** | 所有间距（padding/margin/gap）严格使用 8 的倍数：4/8/12/16/24/32 |
| **Rich Empty States** | 插图 + 语义化标题 + 1-2 句引导 + 主要 CTA 按钮 |
| **Typography Hierarchy** | 4 级字号（32/20/16/14）+ 行高 1.5，不用加粗做层级 |

#### 有赞（同领域直接竞品对标）

| 原则 | 说明 |
|------|------|
| **一屏尽览** | 首页整合核心指标 + 常用工具 + 重要通知，减少菜单跳转 |
| **导航分组清晰** | 业务概览 → 会员 → 门店 → 营销 → 设置，分组语义与美业一致 |
| **卡片间距自定义** | 支持商家自定义卡片间距和热区，个性化和标准化兼顾 |
| **移动端优先** | 手机端导航图标加大、按钮触控区 ≥44px、信息层级精炼 |
| **品牌色贯穿** | 按钮/标签/强调文字统一使用品牌色，形成视觉记忆 |
| **AI 融入 UI** | AI 搜索、AI 生成商品图、AI 分析评价 — 不单独成页，嵌入现有流程 |

#### Stripe Dashboard（排版与层级标杆）

| 原则 | 说明 |
|------|------|
| **阴影代替边框** | 卡片用 `box-shadow` 分层，不使用 `border`，视觉噪音降低 40% |
| **排版即层级** | 仅靠字号+字重+间距区分层级，几乎不用背景色块 |
| **单一工具栏高度** | 所有页面操作栏高度统一为 36px，按钮/输入/选择器都遵循此高度 |
| **渐进式披露** | 表单先显示核心字段，高级选项用"展开"触发 |
| **数据密度适中** | 表格行高 48-52px，每行最多 5-6 列，超出列折叠到详情 |
| **左对齐文本 + 右对齐数字** | 严格遵守，不居中数字 |
