# API 修复策略

> 接口报错时，按本文档步骤排查和修复。

---

## 一、核心架构说明

### 请求链路

```
浏览器 → 前端请求路径 → Vite 代理（/api 前缀转发）→ 后端 Controller 路径
```

### 关键配置

| 配置项     | 位置             | 作用                                               |
| ---------- | ---------------- | -------------------------------------------------- |
| Vite 代理  | `vite.config.js` | 将 `/api` 开头的请求转发到 `http://localhost:8080` |
| Axios 请求 | `request.js`     | 无 baseURL，前端 API 文件直接写完整路径            |
| 后端路径   | Controller 类    | `@RequestMapping("/api/xxx")`                      |

### 路径规则

- **前端 API 文件**中所有请求路径必须以 `/api` 开头
- **后端 Controller** 的 `@RequestMapping` 必须以 `/api` 开头
- **Vite 代理**配置 `'/api'` 前缀转发
- **JWT 拦截器**的路径配置必须与上述路径一致

## 二、接口报错排查流程

### 3.1 登录接口报错

```
登录失败
  ├── 检查浏览器 Network → 请求路径是否正确
  │   ├── 期望：POST /api/auth/login → 200
  │   └── 实际：POST /api/api/auth/login → 404（baseURL 双重前缀）
  ├── 检查后端日志
  │   ├── 有请求日志 → 路径匹配上了
  │   └── 无请求日志 → 路径没匹配上（Vite 代理没转发）
  └── 检查密码
      ├── 数据库密码是 admin123
      └── 前端传的密码也是 admin123
```

### 3.2 其他接口报错

```
接口返回 404/405
  ├── 检查前端 API 路径是否以 /api 开头
  ├── 检查后端 Controller 路径是否以 /api 开头
  ├── 检查 Vite 代理是否配置了 /api 转发
  └── 检查拦截器 excludePathPatterns 是否包含该路径
```

### 3.3 数据解包报错

```
Cannot read properties of undefined
  ├── 检查 Axios 响应拦截器（request.js）
  │   └── res => { const { code, msg, data } = res.data; return data }
  ├── 后端返回结构：{ code: 200, msg: "成功", data: { ... } }
  ├── 拦截器 return data → 前端收到 { ... }
  └── 前端应该用 const result = await apiCall() 而不是 result.data
```

## 三、事故复盘

### 事故 1：API 路径前缀不一致

| 项目     | 内容                                                       |
| -------- | ---------------------------------------------------------- |
| **现象** | 所有接口返回 404                                           |
| **根因** | 后端 Controller 路径无 `/api` 前缀，前端请求有 `/api` 前缀 |
| **教训** | 前后端路径必须在编码前统一约定                             |

### 事故 2：数据解包层级不一致

| 项目     | 内容                                              |
| -------- | ------------------------------------------------- |
| **现象** | `Cannot read properties of undefined`             |
| **根因** | Axios 拦截器已解包，前端再 `.data` 得到 undefined |
| **教训** | 必须先读 Axios 拦截器逻辑，再写前端页面代码       |

### 事故 3：拦截器路径未同步更新

| 项目     | 内容                                                            |
| -------- | --------------------------------------------------------------- |
| **现象** | 登录接口被 JWT 拦截器拦截                                       |
| **根因** | 修改 API 路径后，忘记同步更新拦截器路径                         |
| **教训** | 修改 API 路径时，必须同步检查三处：Controller、前端 API、拦截器 |

### 事故 4：baseURL 导致双重 /api 前缀

| 项目     | 内容                                                                               |
| -------- | ---------------------------------------------------------------------------------- |
| **现象** | 登录接口报错，请求路径变成 `/api/api/auth/login`                                   |
| **根因** | `request.js` 的 `baseURL: '/api'` + 前端 API 文件路径 `/api/auth/login` = 双重前缀 |
| **教训** | 要么用 baseURL（前端路径不加 /api），要么不用 baseURL（前端路径加 /api），不能混用 |
