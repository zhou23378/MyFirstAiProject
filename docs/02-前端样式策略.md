# 前端样式策略

> 每次生成前端样式代码前必须阅读本文档，确保样式架构的健壮性和可维护性。

---

## 一、核心原则

### 1.1 分层架构

```
styles/
├── variables.css          # 第1层：CSS 变量（只定义变量，不写选择器）
├── reset.css              # 第2层：全局重置（reset、字体、滚动条）
├── element-overrides.css  # 第3层：Element Plus 组件覆盖（按组件分类）
└── (无其他全局 CSS 文件)

每个 .vue 文件：
└── <style scoped>         # 第4层：页面/组件专属样式
```

### 1.2 各层职责

| 层         | 文件                            | 职责                            | 修改频率             |
| ---------- | ------------------------------- | ------------------------------- | -------------------- |
| **变量层** | `variables.css`                 | 只定义 CSS 变量，不写任何选择器 | 极低（换主题时才改） |
| **重置层** | `reset.css`                     | 全局 reset、字体、滚动条        | 极低                 |
| **覆盖层** | `element-overrides.css`         | 统一覆盖 Element Plus 组件样式  | 低（统一调整时改）   |
| **组件层** | 各 `.vue` 文件 `<style scoped>` | 页面/组件专属样式               | 高（日常开发）       |

### 1.3 为什么这样设计？

1. **variables.css** — 所有颜色、阴影、圆角都定义为 CSS 变量，后期换主题只需改变量值
2. **element-overrides.css** — 按组件分类（`/* Buttons */`, `/* Tables */`, `/* Dialogs */`），方便查找和修改
3. **scoped 样式** — 每个页面/组件只写自己的样式，互不干扰，删除页面时不会遗留全局样式
4. **不写全局 class** — 避免污染，所有全局样式只针对 Element Plus 的组件类名

## 二、CSS 变量命名规范

### 2.1 变量前缀

所有 CSS 变量以 `--` 开头，按功能分类：

```
--primary-*      # 主色调
--success-*      # 成功色
--warning-*      # 警告色
--danger-*       # 危险色
--info-*         # 信息色
--bg-*           # 背景色
--text-*         # 文字色
--shadow-*       # 阴影
--radius-*       # 圆角
--transition-*   # 过渡动画
--glass-*        # 毛玻璃效果
```

### 2.2 变量定义示例

```css
:root {
  /* 主题色 */
  --primary-color: #6c5ce7;
  --primary-light: #a29bfe;
  --primary-dark: #4834d4;
  --primary-gradient: linear-gradient(135deg, #6c5ce7 0%, #a29bfe 100%);

  /* 背景色 */
  --bg-sidebar: #1a1a2e;
  --bg-main: #f0f2f5;
  --bg-card: #ffffff;

  /* 文字色 */
  --text-primary: #1a1a2e;
  --text-secondary: #636e72;
  --text-sidebar: #a0a4b8;

  /* 阴影 */
  --shadow-sm: 0 1px 3px rgba(0, 0, 0, 0.06);
  --shadow-md: 0 4px 12px rgba(0, 0, 0, 0.08);
  --shadow-lg: 0 8px 24px rgba(0, 0, 0, 0.12);

  /* 圆角 */
  --radius-sm: 6px;
  --radius-md: 10px;
  --radius-lg: 16px;

  /* 过渡 */
  --transition-fast: 0.2s ease;
  --transition-normal: 0.3s ease;
}
```

## 三、Element Plus 覆盖规范

### 3.1 覆盖原则

- 只覆盖 Element Plus 的**组件类名**（如 `.el-button--primary`）
- 不写自定义全局 class
- 按组件分类，用注释分隔
- 每个覆盖规则加注释说明用途

### 3.2 覆盖示例

```css
/* ========================================
   Buttons
   ======================================== */

/* 主按钮：渐变背景 + 发光阴影 */
.el-button--primary {
  background: var(--primary-gradient) !important;
  border: none !important;
  border-radius: var(--radius-sm) !important;
  box-shadow: 0 2px 8px rgba(108, 92, 231, 0.3) !important;
  transition: all var(--transition-fast) !important;
}

.el-button--primary:hover {
  background: var(--primary-gradient-hover) !important;
  box-shadow: 0 4px 16px rgba(108, 92, 231, 0.4) !important;
  transform: translateY(-1px);
}
```

### 3.3 需要覆盖的组件清单

- [ ] Buttons（按钮）
- [ ] Cards（卡片）
- [ ] Tables（表格）
- [ ] Inputs（输入框）
- [ ] Dialogs（弹窗）
- [ ] Pagination（分页）
- [ ] Tags（标签）
- [ ] Dropdown（下拉菜单）
- [ ] Switch（开关）
- [ ] Collapse（折叠面板）
- [ ] Message（消息提示）
- [ ] Select（选择器）
- [ ] InputNumber（数字输入框）
- [ ] DatePicker（日期选择器）
- [ ] MessageBox（确认框）

## 四、页面样式规范

### 4.1 每个 .vue 文件的样式结构

```vue
<style scoped>
/* 页面容器 */
.page-container {
  max-width: 1200px;
  margin: 0 auto;
}

/* 搜索栏卡片 */
.search-card {
  margin-bottom: 16px;
  /* 使用 CSS 变量 */
  border-radius: var(--radius-md);
}

/* 操作栏 */
.action-bar {
  margin-bottom: 16px;
  display: flex;
  justify-content: space-between;
  align-items: center;
}

/* 表格分页 */
.pagination {
  margin-top: 16px;
  display: flex;
  justify-content: flex-end;
}
</style>
```

### 4.2 样式编写规则

1. **必须使用 `scoped`** — 防止样式泄漏到其他组件
2. **使用 CSS 变量** — 所有颜色、阴影、圆角引用 `var(--xxx)`
3. **不写全局选择器** — 不写 `:deep()` 除非必要
4. **类名语义化** — 使用 `.member-table` 而非 `.t1`
5. **避免嵌套过深** — 最多 3 层嵌套
6. **注释分组** — 用注释分隔不同区块

## 五、主题切换方案（预留）

### 5.1 实现方式

未来如需支持主题切换，只需：

1. 定义多套 CSS 变量（如 `:root[theme="dark"]`）
2. 通过 JS 切换 `document.documentElement` 的 `theme` 属性
3. 所有样式自动适配

### 5.2 示例

```css
/* 亮色主题（默认） */
:root {
  --bg-main: #f0f2f5;
  --text-primary: #1a1a2e;
}

/* 暗色主题 */
:root[theme="dark"] {
  --bg-main: #1a1a2e;
  --text-primary: #ffffff;
}
```

## 六、样式文件生成检查清单

每次生成或修改样式文件时，必须逐项确认：

### 6.1 全局样式文件

- [ ] `variables.css` — 是否只定义了 CSS 变量，没有写选择器？
- [ ] `reset.css` — 是否只包含 reset、字体、滚动条？
- [ ] `element-overrides.css` — 是否按组件分类，有注释分隔？

### 6.2 页面样式

- [ ] 是否使用了 `scoped`？
- [ ] 是否引用了 CSS 变量（`var(--xxx)`）而非硬编码值？
- [ ] 类名是否语义化？
- [ ] 是否添加了区块注释？

### 6.3 避免的反模式

- ❌ 在全局 CSS 中写自定义业务 class
- ❌ 在页面样式中写 `!important`（除非覆盖 Element Plus）
- ❌ 硬编码颜色值（应使用 CSS 变量）
- ❌ 在 `element-overrides.css` 中写业务相关样式
