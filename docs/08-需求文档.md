## 三、模块详细需求

### 3.1 登录认证

**功能描述：** 管理员通过账号密码登录，获取 JWT Token 访问其他接口。支持四种角色：ADMIN（超级管理员）、MANAGER（店长）、TECHNICIAN（技师）、CASHIER（收银员）。

**默认账号：** admin / admin123（ADMIN 角色）

**业务规则：**
- 密码使用 BCrypt 加密存储
- Token 有效期 24 小时，内含 role claim
- 所有 `/api/**` 接口（除登录外）都需要 Bearer Token
- 前端根据 role 过滤菜单（menuPermissions 映射）
- 登录限流：5 次/分钟/IP

**数据表：`admin`**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| username | VARCHAR(50) | 用户名（UNIQUE） |
| password | VARCHAR(255) | BCrypt 加密密码 |
| role | VARCHAR(20) | 角色：ADMIN/MANAGER/TECHNICIAN/CASHIER |

### 3.3 会员管理

**功能描述：** 会员信息的增删改查，支持高级筛选、RFM 分层展示、自动标签、导入导出。

**数据表：`member`**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| name | VARCHAR(50) | 会员姓名 |
| phone | VARCHAR(20) | 手机号 |
| gender | TINYINT | 性别（0未知/1男/2女） |
| level | INT | 会员等级ID（关联 member_level.id） |
| points | INT | 积分 |
| balance | DECIMAL(10,2) | 余额 |
| birthday | DATE | 生日（用于自动营销） |
| last_visit_date | DATE | 最后到店日期（消费时自动更新） |
| total_spent | DECIMAL(10,2) | 累计消费金额 |
| visit_count | INT | 到店次数 |
| rfm_segment | VARCHAR(20) | RFM 分层（自动计算） |
| remark | VARCHAR(500) | 备注 |
| create_time | DATETIME | 注册时间 |

**业务规则：**
- 手机号不能重复
- 等级字段关联 `member_level` 表，前端显示等级名称
- 消费后自动累加 total_spent、visit_count、更新 last_visit_date
- RFM 分层每周一自动计算（定时任务）
- 自动标签每日匹配（tag_rule → member_tag）
- 生日当天自动发券 + 短信
- 删除会员前需确认无未完成的订单

**高级筛选：** 支持按姓名/手机号/性别/等级/余额范围/累计消费范围/最后消费天数 组合查询

**页面：**
- 会员列表页：ResponsiveDataList（桌面表格 / 手机卡片）+ 高级筛选 + 导入导出
- 会员详情页：信息卡片 + RFM 风险徽标 + 标签展示 + 消费记录 + 订单明细弹窗

### 3.4 会员等级

**功能描述：** 管理会员等级，不同等级享受不同消费折扣。

**数据表：`member_level`**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| name | VARCHAR(50) | 等级名称 |
| points_required | INT | 升级所需积分 |
| discount_rate | DECIMAL(3,2) | 折扣率（如 0.85 = 85折） |
| sort | INT | 排序号 |
| status | TINYINT | 状态：1=启用 0=停用（DEFAULT 1） |

**默认数据：**

| 等级 | discount_rate | points_required |
|------|---------------|-----------------|
| 普通会员 | 1.00 | 0 |
| 银卡会员 | 0.95 | 200 |
| 金卡会员 | 0.90 | 500 |
| 钻石会员 | 0.85 | 1000 |

**业务规则：**
- 消费时自动应用等级折扣（totalAmount = originalTotal × discountRate）
- 消费后积分达到阈值自动升级（addPointsAndUpgrade）
- 降级暂不支持（需手动操作）

### 3.6 消费收银

**功能描述：** 选择会员 → 按分类选择服务项目 → 购物车结算。支持多种支付方式和组合支付。

**数据表：`consumption_order`**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| member_id | BIGINT | 会员ID |
| total_amount | DECIMAL(10,2) | 折后总金额 |
| pay_method | TINYINT | 支付方式（1现金/2余额/3微信/4支付宝/5银行卡/6储值卡/7团购/8混合） |
| pay_amount | DECIMAL(10,2) | 实付金额 |
| balance_used | DECIMAL(10,2) | 使用余额 |
| points_earned | INT | 获得积分（1元=1积分） |
| employee_id | BIGINT | 服务技师ID |
| commission_amount | DECIMAL(10,2) | 提成金额 |
| status | TINYINT | 状态（1正常/2已退款） |

**数据表：`consumption_order_item`**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| order_id | BIGINT | 订单ID |
| item_id | BIGINT | 服务项目ID |
| item_name | VARCHAR(100) | 项目名称（冗余） |
| item_price | DECIMAL(10,2) | 项目单价（冗余） |
| quantity | INT | 数量 |

**数据表：`payment_detail`**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| order_id | BIGINT | 订单ID |
| pay_method | TINYINT | 支付方式 |
| amount | DECIMAL(10,2) | 金额 |

**业务规则：**
- 自动应用会员等级折扣率
- 支持优惠券抵扣
- 余额支付时自动扣减会员余额
- 消费后按折后金额增加积分（1元=1积分）
- 组合支付（payMethod=8）时写入 payment_detail 表记录各通道明细
- 退款时：退还余额 + 扣回积分 + 订单状态→REFUNDED

**页面：** 三栏布局（会员选择 → 服务项目 → 购物车结算），移动端改为步骤流程。

**功能描述：** 实时展示所有技师的工作状态（空闲/忙碌/休息/下班）、当前服务的客人和项目、服务倒计时。10 秒自动刷新。

### 3.15 轮牌排队（Phase 7 新增）

**功能描述：** 到店顾客取号排队，前台分配空闲技师。FIFO 规则，排队号每天从 1 开始。

**数据表：`service_queue`**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| member_id | BIGINT | 会员ID |
| member_name | VARCHAR(50) | 会员姓名 |
| service_item_id | BIGINT | 期望服务项目ID |
| service_item_name | VARCHAR(100) | 期望服务项目名 |
| queue_number | INT | 排队号（每天重置） |
| status | TINYINT | 1等待中/2已分配/3已取消/4已跳过 |
| assigned_employee_id | BIGINT | 分配的技师ID |
| wait_since | DATETIME | 开始等待时间 |

**业务规则：**
- 入队：当天最大排队号 +1
- 分配：校验技师 AVAILABLE → 更新技师为 BUSY → 自动创建 ServiceTimer
- 整个 assign 操作在一个 @Transactional 中

**页面：** 两栏布局（排队列表 + 空闲技师网格），点击排队卡片 → 点击技师卡片 → 确认分配

### 3.17 财务日结（Phase 7 新增）

**功能描述：** 每日营业结束后，系统自动汇总当天消费数据（按支付方式分通道），与人工实收对比，差异追踪，锁定归档。

**数据表：`daily_close`**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| close_date | DATE | 日结日期（UNIQUE） |
| system_cash | DECIMAL(10,2) | 系统现金 |
| system_wechat | DECIMAL(10,2) | 系统微信 |
| system_alipay | DECIMAL(10,2) | 系统支付宝 |
| system_balance | DECIMAL(10,2) | 系统余额支付 |
| system_card | DECIMAL(10,2) | 系统储值卡 |
| system_total | DECIMAL(10,2) | 系统合计 |
| manual_cash | DECIMAL(10,2) | 人工现金 |
| manual_wechat | DECIMAL(10,2) | 人工微信 |
| manual_alipay | DECIMAL(10,2) | 人工支付宝 |
| manual_total | DECIMAL(10,2) | 人工合计 |
| diff_amount | DECIMAL(10,2) | 差异金额 |
| status | TINYINT | 0草稿/1已锁定 |
| remark | VARCHAR(255) | 备注 |
| locked_by | VARCHAR(50) | 锁定人 |
| locked_at | DATETIME | 锁定时间 |

**业务规则：**
- 系统汇总：从 consumption_order + payment_detail 按日期聚合
- 差异 = 系统合计 - 人工合计
- 锁定后不可修改（status=1）

**页面：** 日期选择 + 两栏对比（系统汇总只读 / 人工录入可编辑）+ 差异高亮 + 锁定按钮

### 3.18 标签规则（Phase 7 新增）

**功能描述：** 定义自动标签规则（条件组合），每日定时匹配会员并打标。支持手动触发。

**数据表：`tag_rule`**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| name | VARCHAR(50) | 规则名称 |
| tag_name | VARCHAR(30) | 生成的标签名 |
| conditions_json | TEXT | 条件 JSON 数组 |
| enabled | TINYINT | 0禁用/1启用 |

**数据表：`member_tag`**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| member_id | BIGINT | 会员ID |
| tag_name | VARCHAR(30) | 标签名 |
| source | VARCHAR(20) | AUTO/MANUAL |
| rule_id | BIGINT | 来源规则ID |

**条件支持的字段：** last_visit_days、total_spent、visit_count、balance、level

**条件支持的操作符：** `>`、`<`、`>=`、`<=`、`=`、`!=`

**业务规则：**
- 最多 3 个条件 AND 组合
- 每日凌晨 3:00 自动执行
- 同会员同标签不重复（UNIQUE KEY uk_member_tag）

**页面：** 规则 CRUD 表格 + 条件编辑器（字段/操作符/值 三列）+ 启用开关 + 手动执行按钮
