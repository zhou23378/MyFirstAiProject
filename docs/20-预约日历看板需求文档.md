## 一、背景与业务价值

### 1.1 现状问题

当前预约管理页面（`/appointment`）是一个**纯表格列表**，存在以下体验缺陷：

| 问题 | 影响 |
|------|------|
| 表格列表无法直观看到每天各时段预约分布 | 前台无法快速判断某天某个时段是否可接新预约 |
| 技师时间线不可见 | 无法一眼看出哪个技师空闲、哪个技师排满 |
| 预约冲突无法前置发现 | 只能在提交表单后靠后端 `checkTimeConflict` 返回错误才知道冲突 |
| 无日/周视图切换 | FullCalendar 组件已安装在项目中但未被预约模块使用 |
| 牌班考勤（schedule）和预约（appointment）是隔离的两个页面 | 店长无法在一个视图中同时看排班和预约 |

### 1.2 客满满对标

客满满的预约日历看板是其**核心差异化功能**，美业商户日常使用频率最高的页面之一：

| 客满满能力 | 当前项目 | 差距 |
|-----------|---------|------|
| 日/周视图日历 | ❌ 无，仅表格列表 | 核心缺口 |
| 技师列视图（每人一列，看当日时间线） | ❌ 无 | 核心缺口 |
| 可视时段网格（30min/60min 粒度） | ❌ 无 | 核心缺口 |
| 点击空白时段快速创建预约 | ❌ 需点"新增"按钮弹窗 | 操作效率差 |
| 拖拽调整预约时间/技师 | ❌ 无 | 进阶体验 |
| 预约状态颜色编码一目了然 | ⚠️ 表中有 el-tag，但不直观 | 可视化不足 |
| 美团/点评预约自动同步 | ❌ 无 | 外部依赖 |

### 1.3 业务价值

- **前台效率提升 50%+**：从"翻列表 → 判断 → 新增"变为"看板上一眼看到空位 → 点击创建"
- **减少预约冲突**：可视网格上重叠时段即时可见，不需要提交后报错
- **技师工作量可视化**：店长可以均衡分配，避免某技师超负荷
- **降低 No-Show 率**：看板上醒目展示待确认/即将到店的预约

## 二、用户故事

| 角色 | 场景 | 期望 |
|------|------|------|
| **前台收银** | 顾客打电话来预约 | 打开日历看板，看一眼就知道今天下午 3 点哪个技师有空，直接点空格创建 |

## 三、功能范围

### 3.1 核心功能（P0 — 必须交付）

| F1 | **日视图日历看板** | 以时间轴 + 技师列的网格形式展示当日所有预约 |
| F2 | **周视图日历** | 7 天概览，每个格子显示预约数量/简要信息 |
| F3 | **预约卡片可视化** | 每个预约呈现为可辨识的卡片，含会员名、服务项目、状态颜色 |
| F4 | **快速创建** | 点击空白时段 → 弹出简化创建表单（日期/时段自动填入） |
| F5 | **状态流转（在看板上）** | 右键/点击卡片 → 到店/完成/取消/爽约 |
| F6 | **列表/看板切换** | 保留现有表格列表，新增"日历看板"Tab |

### 3.2 增强功能（P1 — 显著提升体验）

| F7 | **冲突可视化** | 同一技师同一时段已有预约时，网格背景变红/灰 |
| F8 | **筛选器** | 按技师筛选、按服务项目筛选、按状态筛选 |
| F9 | **预约详情弹窗** | 点击卡片 → 右侧滑出详情面板（非弹窗遮挡） |
| F10 | **拖拽调整** | 拖拽卡片到其他技师/时段 = 修改预约（自动校验冲突） |

### 3.3 远期（P2 — 按需）

| F11 | 预约自动提醒状态标签（已提醒/未提醒） |
| F12 | 打印当日预约清单 |
| F13 | 月视图（月度预约热力图） |

### 4.2 日视图（核心视图）

**时间轴**：08:00 - 21:00（营业时间），30 分钟粒度，可滚动。

**列布局**：
- 第 1 列：时间轴（固定，不滚动）
- 第 2-N 列：每个在职技师一列
- 列宽：桌面 ≥ 140px/列，平板 ≥ 100px/列
- 技工列头部：头像 + 姓名 + 状态指示器（复用 technician_status）

**预约卡片**：
```
┌─────────────────┐
│ 🔵 张三         │  ← 会员名 + 状态颜色条（左侧 3px border）
│    剪发 ¥68     │  ← 服务项目 + 价格
│    备注: ...    │  ← 可选备注
├─────────────────┤
│ 1h 30m          │  ← 卡片高度 = 时长比例（30min ≈ 40px）
└─────────────────┘
```

**状态颜色编码**：

| 状态 | 颜色 | CSS 变量 | 卡片效果 |
|------|------|---------|---------|
| 已预约 | 蓝色 `#409EFF` | `--primary-color` | 蓝色左边框 + 浅蓝背景 |
| 已到店 | 绿色 `#67C23A` | `--success-color` | 绿色左边框 + 浅绿背景 |
| 已完成 | 灰色 `#909399` | `--info-color` | 灰色左边框 + 浅灰背景 |
| 已取消 | 橙色 `#E6A23C` | `--warning-color` | 橙色左边框 + 浅橙背景 + 删除线 |
| 爽约 | 红色 `#F56C6C` | `--danger-color` | 红色左边框 + 浅红背景 |

### 4.3 周视图

- 7 列（周一至周日），不展示时间轴
- 每个格子 = 一天，顶部显示日期 + 星期 + 预约总数
- 格子内显示预约简要列表（会员名 + 服务 + 时间），超过 3 条显示"+N 条"
- 点击日期格子 → 切换到该日的日视图
- 当日高亮边框

### 4.4 响应式断点

| 断点 | 日视图 | 周视图 |
|------|--------|--------|
| ≥ 1024px | 完整时间轴 + 全部技师列，横向滚动 | 7 列完整 |
| 768-1023px | 时间轴宽度收窄，技师列减小，横向滚动 | 7 列紧凑 |
| < 768px | 竖屏改为"今日时间线"（列表式，按时间排列，显示技师名），不可用网格 | 3 列 + 左右滑动 |

### 4.5 交互定义

| 操作 | 触发方式 | 行为 |
|------|---------|------|
| **快速创建** | 点击空白时段格子 | 弹出创建 Dialog，日期+开始时间自动填入，技师自动填入 |
| **查看详情** | 点击预约卡片 | 右侧滑出详情面板（Drawer），含完整信息 + 操作按钮 |
| **状态变更** | 卡片上右键菜单 / 详情面板操作按钮 | 调用现有 `updateStatus` API |
| **拖拽移动** | 拖拽卡片到其他技师列/时段 | `PUT /api/appointment/{id}` + 冲突校验 |
| **切换日期** | ← → 箭头 / 日期选择器 | 重新加载对应日/周数据 |
| **切换视图** | 日/周切换按钮 | 保留筛选条件，只切换视图模式 |

### 5.1 新增 API

### 5.2 复用现有 API（无需修改）

| API | 用途 |
|-----|------|
| `POST /api/appointment` | 快速创建（从日历点击触发的表单提交） |
| `PUT /api/appointment/{id}` | 拖拽修改预约（时间/技师变更） |
| `PUT /api/appointment/{id}/status` | 状态流转（到店/完成/取消/爽约） |

| `POST /api/appointment/{id}/convert-to-order` | 转消费订单 |
| `DELETE /api/appointment/{id}` | 删除预约 |
| `GET /api/technician-status` | 获取所有技师当前状态（用于列头指示器） |

### 5.3 现有 API 需要增强

**`GET /api/appointment/page`** — 无需修改。列表视图继续使用此接口。

## 六、数据库变更

**不需要新增表或字段。** 预约日历看板是纯展示层功能，所有数据来自现有表：

- `appointment` — 预约数据
- `employee` — 技师信息
- `technician_status` — 技师实时状态（列头指示器）
- `member` — 会员姓名
- `service_item` — 服务项目名称/价格

`appointment` 表已有 `employee_id`、`appointment_date`、`start_time`、`end_time` 字段，可直接支持日/周视图的时间轴定位。

**可选增强**（P2）：`appointment` 表新增 `color` 字段（VARCHAR(20) NULL），允许手动标记预约卡片颜色（如红色=VIP客户）。本期不实现。

## 八、用户交互流程

### 8.1 主流程：前台创建预约

```
打开日历看板（默认日视图，默认当天）
        ↓
浏览技师列，找到空闲时段（空白格子）
        ↓
点击空白格子 → 弹出创建 Dialog
  ├── 日期/开始时间/技师 自动填入
  ├── 选择会员（搜索）
  ├── 选择服务项目（自动填入时长 → 自动计算 endTime）
  ├── 填写备注
  └── 点击"确认创建"
        ↓
POST /api/appointment → 成功后刷新日视图
        ↓
新预约卡片出现在对应技师列的正确时段位置
```

### 8.2 状态流转流程

```
点击预约卡片 → 右侧滑出详情面板
        ↓
详情面板显示：会员信息、服务项目、时间、技师、备注、当前状态
        ↓
操作按钮（根据当前状态显示）：
  已预约 → [确认到店] [取消预约] [标记爽约] [转消费]
  已到店 → [完成服务] [标记爽约] [转消费]
  已完成 → （只读，无可操作按钮）
  已取消 → （只读）
  爽约   → （只读）
```

### 8.3 拖拽调整流程（P1）

```
拖拽预约卡片
  ├── 水平拖到其他技师列 → 修改技师
  ├── 垂直拖到其他时段 → 修改开始时间
  └── 释放
        ↓
PUT /api/appointment/{id} (body: { employeeId, startTime, endTime })
        ↓
成功 → 卡片动画移动到新位置
失败（冲突）→ 卡片弹回原位 + ElMessage.warning("该时段技师已有预约")
```

### 依赖与风险

| 依赖/风险 | 说明 | 缓解 |
|----------|------|------|
| FullCalendar 不适用 | 技师列+时间轴双维度在 FullCalendar 中实现成本高 | 自研 CSS Grid，更灵活可控 |
| 大量预约性能 | 某日 50+ 预约 × 15 技师时 DOM 节点较多 | 虚拟滚动（按需引入）或分页加载 |
| 拖拽实现复杂度 | 原生 Drag API 在移动端支持差 | P1 先桌面端，移动端不做拖拽 |
