项目启动指南：

启动步骤（使用系统 JDK）
前置检查

java -version     # JDK 21+
mvn -version      # Maven 3.9+
node -v           # Node.js 18+
mysql -V          # MySQL 8.x 运行中
第一步：初始化数据库（仅首次）

mysql -uroot -p123456 -e "CREATE DATABASE IF NOT EXISTS salon DEFAULT CHARACTER SET utf8mb4;"
mysql -uroot -p123456 salon < backend/src/main/resources/schema.sql

第二步：启动后端（8080）

cd backend
mvn compile -q
mvn spring-boot:run
验证：


curl -s -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
# 返回 JWT token 即成功

第三步：启动前端（3000）

cd frontend
npm install          # 仅首次
npm run dev
浏览器访问 http://localhost:3000，登录 admin / admin123。

第四步（可选）：启动 AI 服务（8000）

pip install -r requirements_ai.txt    # 仅首次
cd ai
uvicorn server:app --host 127.0.0.1 --port 8000

# 首次或数据更新后重建向量库
python embed_from_db.py
关键信息：

数据库地址：localhost:3306，库名 salon，账号 root / 123456
后端端口：8080，Swagger：http://localhost:8080/swagger-ui/index.html
前端端口：3000，管理端登录 admin / admin123
顾客端 H5：http://localhost:3000/h5/login

**4 个容器，1 个网络：**

| 容器 | 镜像 | 端口 | 健康检查 |
|------|------|------|----------|
| `salon-mysql` | `mysql:8.0` | 3306 | `mysqladmin ping` |
| `salon-backend` | `Dockerfile.backend` | 8080 | `/actuator/health` |
| `salon-frontend` | `Dockerfile.frontend` | 80 | HTTP 200 |
| `salon-ai` | `Dockerfile.ai` | 8000 | `/api/ai/health` |

**依赖链：** frontend → backend → mysql ← ai-python

## 二、Dockerfile 设计

### 2.1 后端 — 多阶段构建

```dockerfile
# Stage 1: Maven + JDK 21 构建
FROM maven:3.9-eclipse-temurin-21-alpine AS build
WORKDIR /app
COPY backend/pom.xml .
RUN mvn dependency:go-offline -q
COPY backend/src ./src
RUN mvn package -DskipTests -q

# Stage 2: 最小 JRE 运行时 (~200MB)
FROM eclipse-temurin:21-jre-alpine
WORKDIR /app
COPY --from=build /app/target/*.jar app.jar
HEALTHCHECK --interval=30s CMD wget -qO- http://localhost:8080/actuator/health
ENTRYPOINT ["java", "-jar", "app.jar"]
```

### 2.2 前端 — 多阶段构建

```dockerfile
FROM node:20-alpine AS build
WORKDIR /app
COPY frontend/package.json .
RUN npm install
COPY frontend .
RUN npm run build

FROM nginx:alpine
COPY nginx.conf /etc/nginx/conf.d/default.conf
COPY --from=build /app/dist /usr/share/nginx/html
```

### 2.3 AI 服务

```dockerfile
FROM python:3.11-slim
RUN pip install --no-cache-dir chromadb sentence-transformers pymysql fastapi uvicorn
RUN python -c "from sentence_transformers import SentenceTransformer; \
    SentenceTransformer('BAAI/bge-small-zh-v1.5')"
COPY ai/ /app/ai/
CMD ["uvicorn", "ai.server:app", "--host", "0.0.0.0", "--port", "8000"]
```

## 三、docker-compose.yml 关键设计

| 决策 | 选择 | 原因 |
|------|------|------|
| MySQL 基础镜像 | `mysql:8.0`（官方） | 官方维护、安全补丁及时 |
| 前端托管 | Nginx（非 `npm run dev`） | 生产级性能，gzip，API 代理 |
| 网络模式 | 默认 bridge | 容器间服务名 DNS 互访 |
| 日志 | stdout/stderr | Docker 统一收集 |
| 健康检查 | `condition: service_healthy` | 确保启动顺序正确 |

**环境变量流向：**

```
.env 文件
  ├── DB_PASSWORD ──► mysql.MYSQL_ROOT_PASSWORD → backend / ai-python
  ├── JWT_SECRET ───► backend.JWT_SECRET
  └── *_PORT ───────► 宿主机端口映射
```

## 五、日常操作

```bash
# 启动 / 停止
docker compose up -d              # 后台启动
docker compose down               # 停止（Volume 保留）
docker compose down -v            # 停止并删除数据（⚠️ 不可恢复）

# 重启 / 更新
docker compose restart backend    # 重启单个服务
docker compose build backend      # 重新构建
docker compose up -d --no-deps backend  # 更新部署

# 日志
docker compose logs -f backend    # 实时跟踪
docker compose logs --tail=100    # 最近 100 行
docker compose logs --since 1h    # 最近 1 小时

# 进入容器
docker compose exec backend /bin/sh
docker compose exec mysql mysql -uroot -p${DB_PASSWORD} salon
```

## 六、数据库备份与恢复

### 6.1 备份

```bash
# 手动备份（推荐脚本）
./scripts/backup-db.sh

# Docker 内直接备份
docker compose exec mysql mysqldump -uroot -p${DB_PASSWORD} salon | gzip > backup.sql.gz

# 定时备份（crontab，每日凌晨 2 点）
0 2 * * * /app/scripts/backup-db.sh >> /var/log/salon-backup.log 2>&1
```

备份存储：`./backups/salon_YYYYMMDD_HHMMSS.sql.gz`，默认保留 30 天。

### 6.2 恢复

```bash
# 列出可用备份
./scripts/restore-db.sh

# 从指定文件恢复
gunzip -c backup.sql.gz | docker compose exec -T mysql mysql -uroot -p${DB_PASSWORD} salon
```

## 七、换电脑迁移

### 7.1 旧电脑

```bash
# 1. 推送代码
git add -A && git commit -m "backup" && git push origin master

# 2. 导出数据库
mysqldump -uroot -p123456 --databases salon \
  --routines --triggers --single-transaction \
  > salon_backup_$(date +%Y%m%d).sql

# 3. 复制 .sql 文件到 U 盘/网盘
```

### 7.2 新电脑

```bash
# 1. 拉取代码
git clone https://gitee.com/zhou-jie2570/my-first-ai-project.git
cd my-first-ai-project

# 2. 配置环境
cp .env.example .env

# 3. 构建 + 启动（首次 5-15 分钟）
docker compose up -d --build

# 4. 导入数据库
docker exec -i salon-mysql mysql -uroot -psalon123 salon < salon_backup_20260519.sql

# 5. 重建 AI 向量库
docker exec salon-ai python ai/embed_from_db.py

# 6. 验证
curl http://localhost:8080/actuator/health    # → UP
curl http://localhost:8000/api/ai/health      # → {"status":"ok"}
curl -s -o /dev/null -w "%{http_code}" http://localhost/  # → 200
```

### 7.3 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `docker compose` 报错 | Docker 版本旧 | 改用 `docker-compose`（有连字符） |
| 端口冲突 | 本地程序占用 | 修改 `.env` 端口映射 |
| MySQL 启动失败 | 裸机 MySQL 占用 3306 | `net stop MySQL` 或改 MYSQL_PORT |
| 前端除首页外 404 | nginx.conf 未挂载 | 检查根目录 nginx.conf |
| AI 搜索无结果 | 向量库未重建 | 执行步骤 5 |

## 八、监控与健康检查

| 端点 | 用途 |
|------|------|
| `GET /actuator/health` | 后端健康（含 DB 连接） |
| `GET /api/ai/health` | AI 服务健康 |
| `GET /` | 前端正常响应 |

**日志格式：** 每条请求自动注入 `traceId`（8 位 UUID），响应头返回 `X-Trace-Id`。

```bash
# 数据库连接数
docker compose exec mysql mysql -uroot -p${DB_PASSWORD} -e "SHOW PROCESSLIST;"

# 资源使用
docker stats salon-backend --no-stream
df -h ./backups
```

## 九、故障排查

| 症状 | 排查 | 常见原因 |
|------|------|---------|
| 服务无法启动 | `docker compose logs backend` | MySQL 未就绪、端口冲突、内存不足 |
| 数据库连接失败 | `docker compose exec backend wget -qO- http://mysql:3306` | MySQL 密码不一致 |
| AI 服务异常 | `curl http://localhost:8000/api/ai/health` | ChromaDB 数据损坏 |
| 前端 404 | `docker compose exec frontend nginx -t` | nginx.conf 配置错误 |

## 十、环境变量参考

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DB_PASSWORD` | salon123 | MySQL root 密码 |
| `JWT_SECRET` | change-me... | JWT 签名密钥（生产必改） |

| `MYSQL_PORT` | 3306 | MySQL 宿主机端口 |
| `BACKEND_PORT` | 8080 | 后端 API 端口 |
| `FRONTEND_PORT` | 80 | 前端 Web 端口 |
| `AI_PORT` | 8000 | AI 服务端口 |
| `CORS_ORIGINS` | http://localhost:80,... | 允许的跨域来源 |
