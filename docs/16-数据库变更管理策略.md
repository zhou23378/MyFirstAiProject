# 数据库变更管理策略

> 本文档定义数据库结构变更的规范。当前使用纯 SQL 脚本初始化，无 Flyway/Liquibase。

---

## 一、现状

| 项 | 状态 |
|----|------|
| 初始化方式 | `schema.sql`（手动执行） |
| 版本管理 | **无** — 无 Flyway、无 Liquibase、无增量脚本 |
| 迁移工具 | 无依赖 |
| SQL 模式 | `spring.sql.init.mode` 未配置 |

**当前 `schema.sql` 特点：**
- 全部使用 `CREATE TABLE IF NOT EXISTS`（幂等，可重复执行）
- 种子数据使用 `INSERT IGNORE`（幂等，不重复插入）
- MySQL 8 不支持 `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` / `CREATE INDEX IF NOT EXISTS`；补列和补索引需用 `information_schema` 检查 + 动态 SQL 保持幂等
- 无版本号、无变更记录

---

## 二、变更规范（当前阶段）

### 2.1 新增表

1. 在 `schema.sql` 末尾追加 `CREATE TABLE IF NOT EXISTS` 语句
2. 同时在本文档的"变更记录"章节追加一条记录
3. 手动在 MySQL 中执行该 SQL

**SQL 模板：**
```sql
-- [变更日期] [变更人] [变更说明]
CREATE TABLE IF NOT EXISTS new_table (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键',
    name        VARCHAR(50) NOT NULL COMMENT '名称',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='表说明';
```

### 2.2 新增字段

1. 在 `schema.sql` 中修改对应的 `CREATE TABLE` 语句（保持 `IF NOT EXISTS`）
2. **同时**编写独立的 `ALTER TABLE` 语句，在已存在的数据库上执行
3. 在变更记录中追加说明

**示例：**
```sql
-- 在已存在的 member 表上加新字段
SET @sql = IF(
  EXISTS (
    SELECT 1 FROM information_schema.columns
    WHERE table_schema = DATABASE()
      AND table_name = 'member'
      AND column_name = 'remark'
  ),
  'SELECT 1',
  'ALTER TABLE member ADD COLUMN remark VARCHAR(200) COMMENT ''备注'''
);
PREPARE stmt FROM @sql; EXECUTE stmt; DEALLOCATE PREPARE stmt;
```

### 2.3 修改字段

禁止直接修改已有字段类型（有数据时危险）。替代方案：
1. 新增一个字段
2. 迁移数据
3. 废弃旧字段（后续版本删除）

### 2.4 种子数据

```sql
INSERT IGNORE INTO table_name (col1, col2) VALUES ('val1', 'val2');
```

**原则：** 种子数据只增不改。修改已有种子数据用 `UPDATE`。

## 四、未来迁移方案（当增量脚本足够多时启用）

### 4.1 触发条件

当满足以下任一条件时，引入 Flyway：
- 增量 SQL 脚本超过 5 个
- 需要多环境同步（dev/test/prod）
- 第一次部署到非开发环境

### 4.2 迁移步骤

1. 在 `pom.xml` 添加 Flyway 依赖
2. 将现有 `schema.sql` 重命名为 `V1__init.sql`
3. 后续变更写为 `V2__xxx.sql`、`V3__xxx.sql` ...
4. 在 `application.yml` 启用 Flyway

### 4.3 Flyway 目录规范

```
backend/src/main/resources/
└── db/migration/
    ├── V1__init.sql              # 初始建表
    ├── V2__add_member_level.sql  # 会员等级
    └── V3__add_xxx.sql           # 后续变更
```
