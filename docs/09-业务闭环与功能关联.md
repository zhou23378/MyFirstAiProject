## 二、七大业务闭环

### 闭环 1：会员生命周期

```
录入会员 → 消费累积积分 → 自动升级等级 → RFM 分层 → 自动标签 → 精准营销 → 到店消费 → 循环
                ↑                                                    │
                └──────── 生日自动发券 + 沉睡唤醒短信 ←──────────────┘
```

**涉及表（7 张 + member.rfm_segment 字段）：** member → member_level → consumption_order → coupon → tag_rule → member_tag → notification_log → rfm_segment（member 字段）

**关键节点：**
- 消费时自动累加 `total_spent`、`visit_count`、更新 `last_visit_date`
- 每周一凌晨 2:00 RFM 定时任务重新分层（8 段）
- 每日凌晨 3:00 标签引擎遍历规则匹配会员 → 自动打标
- 每日 8:00 生日定时任务：当天生日会员 → 自动发券 + 发短信
- 积分达到等级阈值 → 消费时自动触发 `addPointsAndUpgrade()`

**闭环状态：** ✅ 完整闭环

---

### 闭环 2：消费 → 支付 → 对账

```
选择会员 → 选择服务/商品 → 应用等级折扣 → 应用优惠券 → 多方式支付 → 订单完成
                                                                    ↓
                                                        财务日结（系统汇总 vs 人工实收）
                                                                    ↓
                                                        差异追踪 → 锁定归档（不可修改）
```

**涉及表（4 张）：** consumption_order → consumption_order_item → payment_detail → daily_close

**资金链路（5 通道独立核算）：**

| 通道 | 字段 | 来源 |
|------|------|------|
| 现金 | `system_cash` | pay_method = 1 |
| 微信 | `system_wechat` | pay_method = 3 |
| 支付宝 | `system_alipay` | pay_method = 4 |
| 余额 | `system_balance` | balance_used |
| 储值卡 | `system_card` | pay_method = 6 |

**对账逻辑：** `DailyCloseServiceImpl.getTodaySummary()` → JdbcTemplate 聚合当天消费订单 → 分通道汇总 → 与人工实收对比 → 差异 = 系统合计 - 人工合计

**闭环状态：** ✅ 完整闭环（支付为模拟，真支付待 Phase 8）

---

### 闭环 3：预约 → 到店 → 服务

```
管理员创建预约 → 确认时段/技师 → 顾客到店 → 轮牌排队入队 → 分配技师 → 服务计时 → 完成消费
    ↑                                                                              │
    └──────────────── 消费记录回写 appointment 状态 ←──────────────────────────────┘
```

**涉及表（5 张）：** appointment → service_queue → technician_status → service_timer → consumption_order

**状态联动链：**
```
appointment(status=1 已预约) → 到店(status=2) → 入队 → 分配技师 → 计时开始 → 完成(status=3)
```

**闭环状态：** ⚠️ 管理端闭环完成，缺顾客自助预约入口（Phase 8）

---

### 闭环 4：技师 → 排队 → 计时（店务核心）

```
技师上班(AVAILABLE) → 会员入队取号 → 前台分配技师 → 技师变 BUSY
                                                        ↓
                                                  计时器开始倒计时
                                             80% 黄色预警 / 100% 红色超时
                                                        ↓
                                                  服务完成 → 技师恢复 AVAILABLE → 接下一单
```

**涉及表（3 张）：** technician_status ↔ service_queue ↔ service_timer

**状态机三方联动（@Transactional 保证一致性）：**

```
ServiceQueue(waiting) + TechnicianStatus(AVAILABLE)
        ↓ assign()
ServiceQueue(assigned) + TechnicianStatus(BUSY) + ServiceTimer(running)
        ↓ complete()
ServiceQueue(completed) + TechnicianStatus(AVAILABLE) + ServiceTimer(completed)
```

**关键约束：**
- 只有 AVAILABLE 技师可被分配
- 计时期间技师不可接新单（BUSY 状态阻塞）
- 暂停/恢复正确修正计时（暂停期间不计入 actualDuration）

**闭环状态：** ✅ 完整闭环

---

### 闭环 5：库存进出

```
供应商 → 采购入库(stock_record type=1) → product.stock_qty 增加
                                                ↓
                    服务消耗出库(type=2) / 零售出库(type=2) → stock_qty 减少
                                                ↓
                                    stock_qty <= alert_qty → 库存预警 → 补货
```

**涉及表（3 张）：** product → stock_record → supplier

**闭环状态：** ✅ 完整闭环

---

### 闭环 6：营销自动化

```
RFM 分层(每周一 2:00) → 标签规则匹配(每日 3:00) → 会员分群打标
                                                    ↓
                                    生日/节日/沉睡唤醒 自动触发
                                                    ↓
                                    短信通知 + 优惠券自动发放
                                                    ↓
                                    到店消费 → RFM 数据更新 → 循环
```

**涉及表（5 张）：** tag_rule → member_tag → notification_log → coupon → member

**定时任务一览：**

| 任务 | Cron | 功能 |
|------|------|------|
| BirthdayScheduler | `0 0 8 * * *` | 每天 8:00 查当天生日会员 → 发券 + 短信 |
| RfmScheduler | `0 0 2 * * 1` | 每周一 2:00 计算 R/F/M → 8 段分层 |
| TagRuleScheduler | `0 0 3 * * *` | 每天 3:00 遍历标签规则 → 匹配 → 打标 |

**RFM 8 段分层：** CHAMPIONS / LOYAL / POTENTIAL / NEW / AT_RISK / NEED_ATTENTION / DORMANT_HIGH / LOST

**闭环状态：** ✅ 完整闭环

## 四、关键数据流（营业核心链路）

```
member ──→ appointment ──→ service_queue ──→ service_timer ──→ consumption_order ──→ daily_close
  │             │               │                  │                  │
  └── level ────┘               └── technician_status ─┘                  │
                                                                          ↓
                              payment_detail ──→ report ──→ AI
```

**一条完整的到店消费链路：**
1. 会员已注册（member 表）
2. 可选：提前预约（appointment 表）
3. 到店 → 入队取号（service_queue 表）
4. 分配空闲技师（technician_status 状态变更）
5. 开始服务计时（service_timer 表）
6. 服务完成 → 创建消费订单（consumption_order 表）
7. 多方式支付 → 记录支付明细（payment_detail 表）
8. 日终对账（daily_close 表）

## 五、权限体系

| 角色 | 可访问路由 | 说明 |
|------|-----------|------|
| **ADMIN** | 全部 17 条 | 超级管理员，可管理系统配置 |
| **MANAGER** | 16 条（除 /admin） | 店长，可管理所有业务但不可改系统 |
| **TECHNICIAN** | 5 条（仪表盘/预约/排班/会员/技师看板） | 技师，只看自己相关 |
| **CASHIER** | 5 条（仪表盘/会员/消费/优惠券/排队） | 收银员，前台操作 |

**认证方式：** JWT Bearer Token，24h 有效期，BCrypt 密码存储。Token 内含 `role` claim，前端根据 role 过滤菜单。

## 六、跨模块耦合度

```
consumption ──── member ──── memberlevel ────  最重耦合（订单依赖会员+等级）
technician ───── queue ───── timer ──────────  店务三模块强耦合（@Transactional 联动）
dashboard ────── JdbcTemplate(跨表SQL) ───────  中度耦合（统计聚合）
report ───────── JdbcTemplate(跨表SQL) ───────  中度耦合（统计聚合）
appointment ──── member, service, employee ────  轻度耦合（读关联）
coupon ───────── member ──────────────────────  轻度耦合
schedule ─────── employee ────────────────────  轻度耦合
其余模块 ────────────────────────────────────  独立
```

**迁移风险排序（高→低）：** consumption > technician/queue/timer > member > dashboard/report > 其余
