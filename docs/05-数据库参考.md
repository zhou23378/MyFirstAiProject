### 4.7 提成结算（F5）

**commission_settlement** — 提成结算单

| 字段 | 类型 | 说明 |
|------|------|------|
| employee_id | BIGINT NOT NULL | 技师ID |
| employee_name | VARCHAR(50) NOT NULL | 技师姓名（冗余） |
| period_start | DATE NOT NULL | 结算周期开始 |
| period_end | DATE NOT NULL | 结算周期结束 |
| order_count | INT DEFAULT 0 | 关联订单数 |
| total_commission | DECIMAL(12,2) DEFAULT '0.00' | 提成总额 |
| status | TINYINT DEFAULT 0 | CommissionSettlementStatus: 0=草稿 1=已确认 2=已支付 |
| confirmed_at | DATETIME | 确认时间 |
| paid_at | DATETIME | 支付时间 |
| remark | VARCHAR(500) | 备注 |

索引：idx_employee, idx_period (period_start, period_end), idx_status

建表 DDL：
```sql
CREATE TABLE commission_settlement (
  id bigint AUTO_INCREMENT,
  employee_id bigint NOT NULL,
  employee_name varchar(50) NOT NULL,
  period_start date NOT NULL,
  period_end date NOT NULL,
  order_count int DEFAULT 0,
  total_commission decimal(12,2) DEFAULT '0.00',
  status tinyint DEFAULT 0,
  confirmed_at datetime,
  paid_at datetime,
  remark varchar(500),
  create_time datetime DEFAULT CURRENT_TIMESTAMP,
  update_time datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  KEY idx_employee (employee_id),
  KEY idx_period (period_start, period_end),
  KEY idx_status (status)
);
```

**数据来源：** 聚合 `consumption_order`（status=1, employee_id NOT NULL）按 `employee_id` 分组，汇总 `order_count` 和 `SUM(commission_amount)`。

SELECT t.id, t.name, t.issued_qty AS cached,
       COALESCE(COUNT(o.id), 0) AS actual
FROM group_buy_template t
LEFT JOIN group_buy_order o ON o.template_id = t.id AND o.status != 5
GROUP BY t.id
HAVING cached != actual;
```
