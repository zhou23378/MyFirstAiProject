### 路由守卫逻辑

```
router.beforeEach:
  ├── 无 token 且 非 noAuth → 跳转 /login
  ├── 有 token 且 访问 /login → 跳转 /
  ├── H5 路径 (/h5/*) → 校验 customer_token，无则跳转 /h5/login
  ├── 管理员路径 (/admin) → 校验 role === 'ADMIN'，非管理员跳转 /
  └── 其他 → 放行
```

**Token 存储：** `localStorage.getItem('token')` / `localStorage.getItem('customer_token')`（H5 端）
**角色存储：** `localStorage.getItem('role')`（用于管理员路由守卫）

### request.js 核心行为

```
请求拦截器：
  token 存在 → headers.Authorization = `Bearer ${token}`

响应拦截器（成功）：
  res.data.code === 200 → return res.data.data     ← 解包，隐藏 { code, msg }
  res.data.code !== 200 → ElMessage.error + 清除 token/role/customer_token + 跳登录

响应拦截器（失败）：
  超时 → ElMessage.error('请求超时')
  无响应 → ElMessage.error('网络错误')
  其他 → ElMessage.error('服务器错误')
```

> **注意：** 401 响应会同时清除 localStorage 中的 `token`、`role`、`customer_token` 三个字段。

**关键影响：** 所有 API 方法返回的是 `data` 字段的内容，不是完整的 `{ code, msg, data }`。

## 八、新增页面开发步骤

```
1. 创建页面文件
   └── frontend/src/views/{module}/index.vue

2. 创建 API 文件（如不存在）
   └── frontend/src/api/{module}.js
   └── import request from './request'
   └── 导出 API 对象，路径以 /api 开头

3. 添加路由
   └── router/index.js children 数组
   └── meta: { requiresAuth: true }  ← 默认，登录除外

4. 添加菜单
   └── AppLayout.vue → <el-menu> 中添加 <el-menu-item>
   └── pageTitle map 中添加标题

5. 样式规范
   └── 使用 <style scoped> + var(--xxx) CSS 变量
   └── 不写全局 class，不覆盖已有样式
```
